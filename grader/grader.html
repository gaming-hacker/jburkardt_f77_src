GRADER
Class Grade Record Management.

GRADER is a FORTRAN77 program which is designed for class room grading. The program stores student records, updates them, and produces final grades based on weights.

A spreadsheet program is surely a more sensible choice. At the time, however, GRADER was an enjoyable program to write, and found some small use locally.

GRADER is intended to be useful for an instructor who keeps a 'running' grade sheet for a class. Several tests are given, with grades being entered after each test. Grades may be modified, and students may drop out. At any time, the class and individual averages may need to be produced. For large classes, there may be a subdivision into sections.

On the first use of GRADER for a particular class, GRADER will construct the file from scratch. You may choose to enter only the class title and the names of the students, or you may also include whatever grade information you already have. When starting up a grade file, you also must specify whether Social Security Numbers (SSN) or any alphanumeric student identifying code, are to be recorded for each student. Further, you give this file a name.

Thereafter, any time that you wish to modify the file, you must run GRADER, and specify the file name you gave earlier. GRADER will then offer you a menu of actions, including modification of names or results entered earlier, output of all information, or of a class roster or of the current grades and averages, or the entry of new information for a particular test.

GRADER tries to be very careful with what it assumes is sensitive information. Before it works on a file, it sets aside a backup copy. Before each operation, it makes a temporary backup copy. Therefore, if you accidentally delete a student from the file, you can recover that information if you immediately stop the program, type out the temporary file (GRADER.OLD) to make sure it's right, delete the damaged file and rename GRADER.OLD to the old file name.

In some cases, the damage was done several steps ago during the current run. In that case, you can go back to the backup copy (GRADER.SAV) made before the file was touched. You have then lost only the work you did during the current session.

GRADER can handle several files during one run. It can sort a file based on the student names or grades.

List of Routines:

  • GRADER is a program for maintaining class records.
  • ADDNAM adds one or more names to the class roster.
  • ANONYM creates an "anonymous" grade listing.
  • CAPCHR capitalizes a string.
  • CHRCAT concatenates two strings to make a third string.
  • CHRCHP chops out a portion of a string.
  • CHRCTR extracts a real number form a string.
  • CHRDB1 removes all blanks from a string.
  • CHRDB2 replaces double blanks by one.
  • COLSWAP swaps two columns of the grade file.
  • COMPAR compares a name to a key, and returns the number of matching characters.
  • CREATE creates a file for a class and list of students.
  • DEFFIL opens an existing file, or creates a new one.
  • DELETE deletes from the roster all students whose names match the key NAMTMP.
  • DELFIL will delete a file of a given name.
  • EXHEAP externally sorts a list of items into linear order.
  • FLUSHL flushes a string left.
  • FLUSHR flushes a string right.
  • GRACOM computes the grade data.
  • GRAPRN prints out the data graded by GRACOM.
  • HELLO prints out a brief introductory message.
  • HELP prints out a list of the available commands.
  • HLPVMS is an interactive help system.
  • INIT sets the initial values for certain variables.
  • ISWAP swaps two integer values.
  • LENCHR returns the length of STRING up to the last nonblank character.
  • LEQI compares two strings for equality, disregarding case.
  • MALPHA is TRUE if a string is purely alphabetic.
  • MODIFY can change any item related to a given student.
  • NUMFIL keeps track of FORTRAN file unit numbers.
  • PRAVDS prints out the distribution of total scores.
  • PRINT displays a copy of the current grade file to the screen.
  • PRRANK prints out the students in order of their average.
  • READER reads in the data from a grade file.
  • RENAME copies file FORIG into FCOPY, and deletes FORIG.
  • ROSTER makes a class roster, suitable for recording attendance.
  • RSWAP swaps two real values.
  • SORT sorts the student records by name or test score.
  • TEST records the grades of all the students for a new test.
  • WRITER writes out a copy of the student information.
